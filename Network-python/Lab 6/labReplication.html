Database Replication

Objectives

In this lab you will set your Pi as a MariaDB Master/Slave replication server. You must do this lab with a lab partner.

Task 1

  1. On your laptop, open a browser window and connect to your Pi MariaDB server using phpmyadmin.

  2. Create a database, name it as your raspberry Pi name. This is the master database that will be replicated to your partner slave server.

  3. Add a table students to the database that includes three fields id (autonumber), firstName (varchar(45)), lastname (varchar(45)).

  4. Export the database created as a .sql file.

  5. Add a record to the table students with your first name and last name.

Task 2 Configuring MariaDB Server on Master

You now configure your Pi as the master, start a terminal window and follow these steps:

  1. Using nano, edit the /etc/mysql/my.cnf file. Under a new section [mariadb] , add the following lines, then save and exit. 

    [mariadb]
log-bin        = /var/log/mysql/master-bin
log-bin-index  = /var/log/mysql/master-bin.index
binlog_format  = mixed
server-id      = 1
#replace dbname with the name of the database to replicate, the one created before
replicate-do-db = dbname

  2. Using nano, edit the file /etc/mysql/mariadb.conf.d/50-server.cnf 

    #comment out the bind-address
#bind-address=127.0.0.1

  3. Save the file, exit nano and restart MariaDB. Then start MariaDB, and issue the sql commands 

    sudo service mysql restart
mysql -u root -p
    MariaDB [(none)]> CREATE USER 'slave'@'localhost' IDENTIFIED BY 'SlavePassword';
MariaDB [(none)]> GRANT REPLICATION SLAVE ON *.* TO slave IDENTIFIED BY 'SlavePassword' WITH GRANT OPTION;
MariaDB [(none)]> FLUSH PRIVILEGES;
MariaDB [(none)]> FLUSH TABLES WITH READ LOCK;
MariaDB [(none)]> SHOW MASTER STATUS;

  4. You will see a table like the following. Mark down the File and Position, you will need these to configure the slave. 

    +-------------------+----------+--------------+------------------+
| File              | Position | Binlog_Do_DB | Binlog_Ignore_DB |
+-------------------+----------+--------------+------------------+
| master-bin.000002 |      798 |              |                  |
+-------------------+----------+--------------+------------------+

  5. Back on the Pi Terminal window 

    MariaDB [(none)]> UNLOCK TABLES;
MariaDB [(none)]> exit;

  6. On your laptop, use phpmyadmin to connect to the slave server, your partner mariadb server. You will need to ask your partner to enter the password for the root account.

  7. Create a database on the slave server, name it as your raspberry pi name.

  8. Click on the database name to select it.

  9. Import the database exported from the master server.

Taks 3 Configuring MySQL Server on Slave

To configure the slave, follow these steps with your partner on your partnerâ€™s Pi.

  1. Edit the /etc/mysql/my.cnf file using nano.

  2. Under a new sertion [mariadb] section, add the following lines. Do not forget to replace dbname with the name of the database to be replicated: 

    server-id       = 2
replicate-do-db = dbname
relay-log-index = /var/log/mysql/slave-relay-bin.index
relay-log       = /var/log/mysql/slave-relay-bin

  3. Save your changes, exit then using nano, edit the file /etc/mysql/mariadb.conf.d/50-server.cnf 

    #comment out the bind-address
#bind-address=127.0.0.1

  4. Restart the mysql service. You are only a few steps to begin replicating. Log on to the database and run the following commands in the MariaDB prompt. Pay special attention to the MASTER_LOG_FILE and MASTER_LOG_POS variables, which should match the values returned by SHOW MASTER STATUS above. Also replace MASTER_HOST value with the IP address of the Master server. 

    MariaDB [(none)]> CHANGE MASTER TO
MASTER_HOST='10.50.11.34',
MASTER_USER='slave',
MASTER_PASSWORD='SlavePassword',
MASTER_PORT=3306,
MASTER_LOG_FILE='master-bin.000002',
MASTER_LOG_POS=798,
MASTER_CONNECT_RETRY=10,
MASTER_USE_GTID=current_pos;

  5. To view the slave status and to stop the slave, you can use the following command: 

    MariaDB [(none)]> SHOW Slave STATUS;
MariaDB [(none)]> STOP Slave;
Task 4 Replication at work:

  1. On the master server, either through phpmyadmin or using a terminal window, add a record to the table students.

  2. Verify that the table is also automatically updated in the slave table.

Taks 5 Switch Roles with your partner.

If your Pi was the master, configure it as a slave and vice versa.
